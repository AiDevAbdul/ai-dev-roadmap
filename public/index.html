Day 1–2 · Choose Your Automation Platform
1
Understand your options: Make vs. n8n vs. Zapier
Quick Comparison →
Make (Integromat): User-friendly, drag-drop, 1000+ apps, generous free tier. Best for visual learners. | n8n: Self-hosted, more control, better for custom logic, steeper curve. | Zapier: Most apps, pricey, simple workflows.
For this roadmap: Start with Make — fastest to first automation.
2
Sign up for Make
Visit make.com → Sign up with email → Create free account
3
Understand Make's interface: Scenarios, modules, connections
A Scenario = workflow. Modules = individual actions. Connections = authenticated links to apps like Gmail, Slack.
Day 2–3 · Connect Your First Apps
1
Connect Gmail — allow Make to read/send emails

Connect Slack — for notifications & message posting
In Make: Add Slack module → Create new connection → Install Make's Slack app to your workspace → Authorize.
3
Connect Google Sheets — data storage & sync

Test each connection

Day 4–5 · Add AI: Claude API in a Workflow
1
Get your Anthropic API key
Visit console.anthropic.com → Create API key → Copy it
2
Add HTTP request module to call Claude API
In Make →
Add "HTTP" module → Set Method: POST → URL: https://api.anthropic.com/v1/messages → Headers: Authorization: Bearer YOUR_KEY, Content-Type: application/json
3
Build the request body for Claude
{ "model": "claude-opus-4-7", "max_tokens": 1024, "messages": [ { "role": "user", "content": "{{emailBody}}" } ] }
4
Use Claude's response in your next action
After HTTP module completes, add another Sheets row that includes {{http.data.content[0].text}} — Claude's response

Core AI Concepts: Agents · Subagents · MCP
🤖

AI Agents

Programs that perceive input, decide what to do, and take actions in a loop — using tools, memory, and goals rather than just responding to a single prompt.

🪆

Subagents

Specialized agents spun up by a parent (orchestrator) agent to handle specific tasks in parallel — e.g., one researches, another writes, a third formats and sends.

🔌

MCP (Model Context Protocol)

An open standard that lets AI models connect to external tools, databases, APIs, and services through a unified interface — like USB-C for AI capabilities.

Learning Principles

Build to Solve Real Problems

Every exercise automates an actual agency pain point — no toy projects.

Principle 1

Don't Memorize Code — Direct AI

Learn to give precise instructions, review output, and iterate. The skill is delegation, not syntax.

Principle 2

One Workflow at a Time

Fully automate one process before moving on. Depth beats breadth.

Principle 3

Document Everything

Write brief SOPs for every automation — what it does, why, and how to maintain it.
